JsonFromJaxb Example

This example demonstrates how to use JSON representation of JAXB based resources.

A simple web application is presented with just one resource: flight list. A client can obtain the list or update it using XML or JSON representation of the list.

Contents

The flight list web resource is implemented by com.sun.ws.rest.samples.jsonfromjaxb.resources.FlightList class.

The mapping of the URI path space is presented in the following table:

URI path Resource class HTTP methods
/flights/ FlightList GET, PUT

Running the Example

JsonFromJaxb example runs on Glassfish application server (http://glassfish.dev.java.net) and comes along with Netbeans (open source IDE, http://www.netbeans.org/) project files. So it is easy to run it from Netbeans IDE.

Running on Netbeans 5.5.1

Presuming, you have installed Netbeans 5.5.1 and Glassfish v2.

Building And Starting The JsonFromJaxb Service

Open JsonFromJaxb project in Netbeans IDE. If you do not have your Glassfish V2 application server already registered in Netbeans, you will be asked to resolve missing server problem. Netbeans IDE will guide you to do so.

You can now build and deploy the project by pressing F6 (Run Main Project)

Test Client Running

Open JsonFromJaxb Test Client project located in JsonFromJaxbTestClient subdirecory. Then press F6 (Run Main Project). If you see something similar in your Netbeans window, you successfully deployed and tested the service:

Running Without Netbeans 5.5.1

Presuming, you have installed Glassfish v2. AS_HOME variable should point to your glassfish installation directory.

Building And Starting The JsonFromJaxb Service

You make it from JsonFromJaxb directory by 
% $AS_HOME/lib/ant/bin/ant 
% $AS_HOME/bin/asadmin start-domain
% $AS_HOME/bin/asadmin deploy dist/JsonFromJaxb.war

Test Client Running

% cd JsonFromJaxbTestClient
% $AS_HOME/lib/ant/bin/ant run
