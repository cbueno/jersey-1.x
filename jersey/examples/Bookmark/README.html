Bookmark Example

This example demonstrates how to use JPA in the backend. The example is based on bookmark example from the RESTful Web Services book, which was inspired by the http://del.icio.us/ web service.

A bookmark web application is presented that is capable of maintaining users and their bookmarks.

Contents

The example consists of four web resources implemented by the following:

com.sun.ws.rest.samples.bookmark.resources.UsersResource
The users resource that returns a list of users.
com.sun.ws.rest.samples.bookmark.resources.UserResource
The user resource that has userid, username, e-mail and password.
com.sun.ws.rest.samples.bookstore.resources.BookmarksResource
The bookmarks resource that returns a list of bookmarks for actuall user resource.
com.sun.ws.rest.samples.bookstore.resources.BookmarkResource
The bookmark resource that has a bookmark uri and a long and short description.

The mapping of the URI path space is presented in the following table:

URI path Resource class HTTP methods
/users/ UsersResource GET
/users/{userid} UserResource GET, PUT (used also for creating new users), DELETE
/users/{userid}/bookmarks BookmarksResource GET, POST
/users/{userid}/bookmarks/{bmid} BookmarkResource GET, PUT, DELETE

Running the Example

Bookmark example runs on Glassfish application server (http://glassfish.dev.java.net) and needs a running JavaDB (http://developers.sun.com/javadb/) instance for underlying data.

You have two natural options, which JavaDB installation to use.

  • JavaDB bundled with Netbeans 6.0
  • JavaDB bundled with Glassfish V2

The example comes along with Netbeans 6.0 (open source IDE, http://www.netbeans.org/) project files, so the first option is convenient for people running on Netbeans.

Running on Netbeans 6.0

Presuming, you have installed Netbeans 6.0 with bundled Glassfish V2 server.

Running Without Netbeans 6.0

Presuming, you have installed Glassfish v2. AS_HOME variable should point to your glassfish installation directory.

Building And Starting The Bookmark Service

If .asadminpass file is missing at your home directory, you will need to run the following command to get it created: 

% $AS_HOME/bin/asadmin login

You then build and run the example in just one shot from Bookmark directory by 

% $AS_HOME/lib/ant/bin/ant run-on-glassfish

Notice: You might need to change access rights for the ant command before running it: 

% chmod +x $AS_HOME/lib/ant/bin/ant

Test Client Running

% cd TestClient
% $AS_HOME/lib/ant/bin/ant run
