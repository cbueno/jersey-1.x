Simple JAX-WS Endpoint Example

This example demonstrates how to develop RESTful web services with a JAX-WS Endpoint.

Contents

The example consists of five Java classes, one HTML file and one JPEG image:

com.sun.jersey.samples.jaxws.Main
Contains the main function which creates, publishes and stops the JAX-WS Endpoint.
com.sun.jersey.samples.jaxws.resources.MasterResourceBean
A resource class that produces a form in response to a HTTP GET, the form will be used to exercise the other resources in this sample applicaton.
com.sun.jersey.samples.jaxws.resources.ResourceBean1
A resource class that produces the string "Hello World from resource 1" in response to a HTTP GET.
com.sun.jersey.samples.jaxws.resources.ResourceBean2
A resource class that produces the string "Hello World from resource 2" in response to a HTTP GET.
com.sun.jersey.samples.jaxws.resources.ResourceBean3
A resource class that produces different responses based on either the value of a "rep" query parameter or the "Accept" header of a HTTP GET invocation. The form contained in the index.html file will allow you to specify whether to use either the query parameter or the MimeType to use for the request.
index.html
A static HTML file that is returned by the MasterResourceBean resource.
java.jpg
A JPEG image returned by the ResourceBean3 resource.

The mapping of the URI path space is presented in the following table:

URI path Resource class HTTP methods
/start MasterResourceBean GET
/resource1 ResourceBean1 GET
/resource2 ResourceBean2 GET
/resource3/{arg1}/{arg2} ResourceBean3 GET

Running the Sample

Run the sample as follows:

cd examples/SimpleJAXWSEndpoint
ant run

From a Web browser, visit:

http://127.0.0.1:9998/endpoint/start

Select the resource to test in the form and submit it by hitting the Test Resource button. If you select "Resouce 1" or "Resouce 2", a Test Details section will appear on the form showing you what the requested URL was, any Query string used, the value of the "Accept" header used, the HTTP headers received on the result and the conent of the result. Here is a sample of what you would see if you select "Resource 1": 

            
            Test Details:


            Resource:
            Resource 1

            Request:
            /endpoint/resource1
            Query: 
            


            Accept MimeType: 
            

            Result Headers: 
            Last-Modified: Tue, 09 Jan 2007 20:59:59 GMT Content-Type: text/plain Transfer-Encoding: chunked
            Result: 
            Hello World from resource 1
If you select "Resource 3" a Resouce 3 Inputs section of the form will appear. The Param 1 and Param 2 are URI parameters that will be used in the result of testing the "text/plain" or "application/x-www-form-urlencoded" MimeType. The Selection method: lets you choose whether the "rep" query parameter or the "Accept" HTTP header should be used to retrieve the result. After filling in the Resource 3 Input, hit the Test Resource 3 button to see the Test Details.

You can exercise ResourceBean1 directly by visiting:

http://127.0.0.1:9998/endpoint/resource1

You can exercise ResourceBean2 directly by visiting:

http://127.0.0.1:9998/endpoint/resource2

From a Web browser, you can only test the query parameter portion of ResourceBean3. You will need to provide two URI parameters and the rep query parameter. The following URLs can be used:

  • http://127.0.0.1:9998/endpoint/resource3/ramesh/rajiv?rep=0
  • http://127.0.0.1:9998/endpoint/resource3/dave/steve?rep=1
  • http://127.0.0.1:9998/endpoint/resource3/bill/john?rep=2
  • http://127.0.0.1:9998/endpoint/resource3/marc/paul?rep=3

You can stop the sample by hitting the "return" button from the console that you started the sample from.
