ManagedBeansWebApp Example

This example demonstrates how to develop RESTful web service with a Servlet 2.5 container.

Contents

The example consists of two java classes:

com.sun.jersey.samples.managedbeans.resources.ManagedBeanPerRequestResource
com.sun.jersey.samples.managedbeans.resources.ManagedBeanSingletonResource

The mapping of the URI path space is presented in the following table:

URI path Resource class HTTP method Description
/managed-beans-war/managedbean/per-request ManagedBeanPerRequestResource GET Returns injected value and increments it
/managed-beans-war/managedbean/singleton ManagedBeanSingletonResource GET Returns injected value and increments it

The starting point of the application is the main index.html page. The application consists only from two resources and main page provides links to them. The output is just single integer value which is injected to the Managed Bean from web.xml file: 

    <env-entry>
        <env-entry-name>injectedResource</env-entry-name>
        <env-entry-type>java.lang.Integer</env-entry-type>
        <env-entry-value>1</env-entry-value>
    </env-entry>
As you can see, injected value is "1" and this is exactly what you get when you hit per-request resource. Singleton returns "1" too, but only once - every hit increases value of stored variable (so you can see that is really singleton).

Running the Example

If you are working with Jersey GlassFish update center module installed into your existing GlassFish instance, you will need to follow instructions at the module README file in order to deploy the example.

Otherwise, you can run the example using embedded GlassFish as follows:

run

mvn glassfish:run

From a web browser, visit:

http://localhost:8080/managed-beans-webapp/managedbean/per-request
http://localhost:8080/managed-beans-webapp/managedbean/singleton
